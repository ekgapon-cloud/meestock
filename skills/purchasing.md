# Skill: Purchasing — MEE Stock

ใช้ไฟล์นี้เมื่อทำงานเกี่ยวกับใบสั่งซื้อ (Purchase Order) และกระบวนการจัดซื้อ

## สถานะของ Purchase Order (PO)

| สถานะ | คำอธิบาย |
|-------|----------|
| `DRAFT` | ร่าง PO ยังไม่ยืนยัน |
| `ORDERED` | สั่งซื้อกับผู้ขายแล้ว รอรับสินค้า |
| `PARTIALLY_RECEIVED` | รับสินค้าบางส่วนแล้ว |
| `RECEIVED` | รับสินค้าครบแล้ว |
| `CANCELLED` | ยกเลิก PO |

## Flow การจัดซื้อ

1. สร้าง PO (`DRAFT`) — ระบุ supplier, รายการวัสดุ, จำนวน, ราคาต่อหน่วย
2. ยืนยัน PO (`ORDERED`) — ส่งให้ผู้ขาย
3. เมื่อวัสดุมาส่ง → บันทึกการรับเข้า (`stock/receive`) โดยอ้างอิง PO นี้
   - สร้าง `StockLedger` type `RECEIVE` พร้อม `referenceType: "PurchaseOrder"`, `referenceId: PO.id`
4. หากรับครบตามจำนวนที่สั่ง → อัปเดต PO เป็น `RECEIVED`
5. หากรับไม่ครบ → PO เป็น `PARTIALLY_RECEIVED` จนกว่าจะรับครบหรือปิด PO ด้วยมือ

## จุดเชื่อมกับ Reorder Point

- เมื่อยอดคงเหลือของวัสดุใดต่ำกว่า `reorderPoint` ที่กำหนดไว้ ระบบควรแจ้งเตือน (notification) หรือแนะนำให้สร้าง PO ใหม่
- ฟีเจอร์นี้อาจ implement เป็น background job ที่รันตรวจสอบยอดคงเหลือเป็นระยะ

## กฎสำคัญ

1. ห้ามรับเข้าสต๊อก (RECEIVE) เกินจำนวนที่ระบุใน PO โดยไม่มีการอนุมัติพิเศษ (ป้องกันรับเกินโดยไม่ตั้งใจ)
2. การยกเลิก PO (`CANCELLED`) ทำได้เฉพาะ PO ที่ยังไม่มีการรับเข้าใดๆ เลย (ถ้ารับเข้าไปแล้วบางส่วน ต้องจัดการแยกกรณี)
3. ราคาต่อหน่วยใน PO คือค่าที่ใช้คำนวณ Weighted Average Cost เมื่อรับเข้า (ดู `skills/site-costing.md`)

## ข้อมูลที่ควรมีใน PO

- Supplier (ผู้ขาย): ชื่อ, ข้อมูลติดต่อ
- รายการวัสดุ: materialId, จำนวนที่สั่ง, ราคาต่อหน่วย
- ไซต์งานปลายทาง
- วันที่คาดว่าจะได้รับสินค้า (expected delivery date)
- เอกสารแนบ (ใบเสนอราคา, ใบสั่งซื้อจริง)
