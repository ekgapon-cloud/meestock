# คู่มือสำหรับ ฝ่ายจัดซื้อ (Purchasing)

> บทบาทของคุณ: **สร้างและจัดการใบสั่งซื้อ (PO)** จากซัพพลายเออร์
> เมนูที่คุณใช้: Dashboard · Materials · ใบสั่งซื้อ · รายงาน · บัญชีของฉัน

---

## เริ่มต้นใช้งาน
1. เปิดเว็บระบบ → กรอก **อีเมล** และ **รหัสผ่าน** → กด **เข้าสู่ระบบ**
2. เปลี่ยนรหัสผ่านตัวเอง: กดที่ชื่อตัวเองมุมล่างซ้าย → **บัญชีของฉัน**

---

## หน้าที่หลักของคุณ

### 1) สร้างใบสั่งซื้อ (เมนู ใบสั่งซื้อ → สร้าง)
1. กด **สร้างใบสั่งซื้อ**
2. เลือก **ซัพพลายเออร์**
3. เพิ่มรายการวัสดุ (สแกนบาร์โค้ด / พิมพ์รหัส / กล้องมือถือ) พร้อม:
   - **จำนวนที่สั่ง** (จำนวนเต็ม)
   - **ราคาทุนต่อหน่วย** (ใส่ทศนิยมได้)
4. บันทึก → ใบสั่งซื้ออยู่สถานะ **"ร่าง" (DRAFT)**

### 2) ยืนยันการสั่งซื้อ
- เปิด PO ที่เป็นร่าง → กด **"สั่งแล้ว" (ORDERED)** เมื่อส่งคำสั่งซื้อไปยังซัพพลายเออร์แล้ว
- หลังจากนี้ **เจ้าหน้าที่คลัง**จะเป็นผู้รับของเข้าระบบ (คนละหน้าที่กับคุณ)

### 3) ติดตามสถานะใบสั่งซื้อ
| สถานะ | ความหมาย |
|-------|---------|
| ร่าง (DRAFT) | สร้างไว้ ยังไม่ได้สั่ง |
| สั่งแล้ว (ORDERED) | ส่งคำสั่งซื้อแล้ว รอรับของ |
| รับบางส่วน | คลังรับของมาแล้วบางส่วน (อัตโนมัติ) |
| รับครบ | รับของครบตามยอดสั่ง (อัตโนมัติ) |
| ยกเลิก (CANCELLED) | ยกเลิกใบสั่งซื้อ |

- สถานะ "รับบางส่วน/รับครบ" ระบบคำนวณให้เองจากยอดที่คลังรับจริง
- พิมพ์ใบสั่งซื้อเป็น PDF ได้จากหน้ารายละเอียด

---

## ข้อควรรู้
- **การรับของไม่ใช่หน้าที่คุณ** — เป็นงานของเจ้าหน้าที่คลัง (แยกกันเพื่อการควบคุมภายใน)
- จำนวนสั่งซื้อต้องเป็น**จำนวนเต็ม** แต่**ราคาใส่ทศนิยมได้**
- ทุกใบสั่งซื้อมีเลขที่รันอัตโนมัติ (PO-)

---

## คำถามที่พบบ่อย
**แก้ไข PO หลังสั่งแล้วได้ไหม?** สถานะจะเดินตามลำดับ (ร่าง→สั่งแล้ว→รับ...) หากต้องแก้ให้พิจารณายกเลิกแล้วออกใหม่ หรือประสานผู้ดูแลระบบ
**ของมาเกินยอดสั่ง?** ระบบกันไม่ให้คลังรับเกิน หากจำเป็นต้องปรับ ให้แก้ที่ใบสั่งซื้อ
**สั่งของโดยไม่มี PO ได้ไหม?** ระบบรองรับการ "รับของแบบไม่มี PO" ที่ฝั่งคลัง แต่การจัดซื้ออย่างเป็นทางการควรออก PO
