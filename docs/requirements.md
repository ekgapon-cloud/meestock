# Requirements — MEE Stock

## 1. วัตถุประสงค์ของระบบ

พัฒนาระบบบริหารจัดการสต๊อกวัสดุอุปกรณ์ไฟฟ้าสำหรับงานก่อสร้าง เพื่อ:
- ควบคุมยอดคงเหลือวัสดุให้ถูกต้อง แม่นยำ และตรวจสอบย้อนหลังได้ (auditable)
- ลดความคลาดเคลื่อนระหว่างสต๊อกจริงกับระบบ
- รองรับการทำงานหลายไซต์งานพร้อมกัน โดยควบคุมสิทธิ์การเข้าถึงข้อมูลแยกตามไซต์
- ให้ผู้บริหารเห็นภาพรวมต้นทุนและสถานะสต๊อกผ่าน Dashboard

## 2. กลุ่มผู้ใช้งาน (User Roles) — เบื้องต้น

| Role | คำอธิบาย |
|------|----------|
| Admin | จัดการระบบทั้งหมด, ตั้งค่าสิทธิ์ผู้ใช้ |
| Storekeeper (ผู้ดูแลคลัง) | รับเข้า/เบิกจ่ายวัสดุ, ปรับปรุงสต๊อก |
| Site Engineer / ผู้ควบคุมงาน | ขอเบิกวัสดุ, ดูสถานะการเบิกของไซต์ตน |
| Approver (ผู้อนุมัติ) | อนุมัติคำขอเบิกในแต่ละขั้นตอน |
| Executive / ผู้บริหาร | ดู Dashboard ภาพรวมทุกไซต์งาน |

> รายละเอียด role และสิทธิ์แบบเต็มต้องยืนยันและปรับปรุงเพิ่มเติม (ดู `docs/business-rules.md` และ `skills/security.md`)

## 3. Functional Requirements (MVP)

### 3.1 การจัดการวัสดุ (Material Master)
- เพิ่ม/แก้ไข/ลบรายการวัสดุ (SKU, หน่วยนับ, หมวดหมู่)
- กำหนดจุดสั่งซื้อขั้นต่ำ (reorder point) ต่อวัสดุต่อไซต์

### 3.2 การรับเข้าสต๊อก (Stock Receiving)
- บันทึกการรับวัสดุเข้าคลัง พร้อมอ้างอิงใบสั่งซื้อ (PO)
- แนบเอกสาร/รูปภาพประกอบการรับเข้า

### 3.3 การเบิกจ่ายวัสดุ (Material Issue) — Multi-stage Workflow
- สร้างคำขอเบิก (Request)
- อนุมัติคำขอ (Approve) — อาจมีหลายระดับ
- จ่ายวัสดุจริง (Issue) และตัดสต๊อกผ่าน ledger

### 3.4 ยอดคงเหลือ (Stock Balance)
- คำนวณจาก ledger transaction ทั้งหมด (รับเข้า - เบิกจ่าย - ปรับปรุง)
- แสดงยอดคงเหลือ real-time ต่อวัสดุต่อไซต์งาน

### 3.5 การควบคุมสิทธิ์การเข้าถึง (Access Control)
- แยกสิทธิ์ตาม Role และ Data Visibility (ไซต์งานที่รับผิดชอบ)

### 3.6 รายงานและ Dashboard
- รายงานยอดคงเหลือ, ประวัติการเบิกจ่าย, มูลค่าสต๊อกคงเหลือ
- Executive Dashboard สรุปภาพรวมทุกไซต์งาน

## 4. Non-functional Requirements

- ระบบต้องรองรับผู้ใช้งานพร้อมกันหลายไซต์งาน
- ข้อมูล transaction ต้อง immutable (แก้ไขไม่ได้ ต้องสร้าง adjustment transaction แทน)
- Response time ของหน้าจอหลักไม่ควรเกิน 2 วินาที
- รองรับภาษาไทยในทุกหน้าจอที่ผู้ใช้งานเห็น

## 5. Phased Project Tracker (33 วัน) — สรุปสังเขป

| Phase | ขอบเขต |
|-------|--------|
| Phase 1 | MVP: Material master, Stock receiving, Basic issue workflow |
| Phase 2 | Multi-stage approval workflow, Ledger-based balance เต็มรูปแบบ |
| Phase 3 | Two-dimensional access control |
