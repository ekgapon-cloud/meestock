## 1. การเข้าสู่ระบบ

1. เปิดหน้าเว็บของระบบ ระบบจะพาไปที่หน้า **เข้าสู่ระบบ** อัตโนมัติ
2. กรอก **อีเมล** และ **รหัสผ่าน** ที่ได้รับจากผู้ดูแลระบบ (Admin)
3. กด **เข้าสู่ระบบ**

> **ความปลอดภัย:** ระบบพยายามล็อกอินได้ไม่เกิน 10 ครั้งต่อ 15 นาทีต่อเครื่อง หากใส่รหัสผิดบ่อยเกินจะถูกระงับชั่วคราว รอสักครู่แล้วลองใหม่ (การล็อกอินสำเร็จไม่ถูกนับ)

**ออกจากระบบ:** กดปุ่ม **ออกจากระบบ** ที่มุมล่างซ้ายของแถบเมนู

## 2. บทบาทและสิทธิ์การใช้งาน

ระบบแบ่งสิทธิ์เป็น **2 มิติ** ที่แยกจากกัน:

### 2.1 บทบาทงาน (Role) — กำหนดว่า "ทำอะไรได้"
| Role | หน้าที่หลัก | ทำอะไรได้เป็นพิเศษ |
|------|------------|------------------|
| **REQUESTER** | ผู้ขอเบิก (หน้างาน/ช่าง) | สร้างคำขอเบิกวัสดุ |
| **APPROVER** | ผู้อนุมัติ | อนุมัติ/ปฏิเสธคำขอเบิก |
| **WAREHOUSE** | เจ้าหน้าที่คลัง | รับวัสดุ, จ่ายวัสดุ, โอนย้าย, นับสต๊อก, จัดการหมวดหมู่/วัสดุ |
| **PURCHASING** | ฝ่ายจัดซื้อ | สร้าง/จัดการใบสั่งซื้อ (PO) |
| **EXECUTIVE** | ผู้บริหาร | ดูรายงานและ Dashboard เชิงบริหาร |

### 2.2 ระดับการเข้าถึงข้อมูล (Access Level) — กำหนดว่า "เห็นข้อมูลแค่ไหน"
| Access Level | เห็นอะไร |
|--------------|---------|
| **STAFF** | ข้อมูลปฏิบัติงานทั่วไป **แต่ไม่เห็นข้อมูลต้นทุน/มูลค่า** |
| **MANAGER** | เห็นทุกอย่างรวมถึงต้นทุน รายงานการเงิน และจัดการโครงการได้ |
| **ADMIN** | สิทธิ์สูงสุด เห็นและทำได้ทุกอย่าง รวมถึงจัดการผู้ใช้ |

> **สำคัญ:** ระดับ **STAFF จะไม่เห็นราคาทุน/มูลค่าวัสดุ** ทุกที่ในระบบ — ช่องราคาจะถูกซ่อนไว้ ผู้ที่เห็นต้นทุนได้คือ MANAGER และ ADMIN เท่านั้น
>
> `ADMIN` สามารถข้ามข้อจำกัดด้าน Role ได้ทั้งหมด (ทำได้ทุกอย่างไม่ว่า Role จะเป็นอะไร)

### 2.3 การมองเห็นตามไซต์ (Site Access)
ผู้ใช้แต่ละคนถูกกำหนด "คลัง/ไซต์ที่เข้าถึงได้" ได้ (ยกเว้น ADMIN ที่เห็นทุกคลัง) — จะเห็นเฉพาะสต๊อกและเอกสารของคลังที่ตนมีสิทธิ์เท่านั้น ตั้งค่าได้ในหน้า **ผู้ใช้งาน**

## 3. หน้าจอและเมนูหลัก

เมนูอยู่แถบด้านซ้าย (บนจอมือถือจะยุบเป็นแถวไอคอนเลื่อนแนวนอน)

### 📊 Dashboard
หน้าสรุปภาพรวม จำนวนวัสดุ คำขอที่รออนุมัติ ฯลฯ (ผู้บริหาร/MANAGER/ADMIN เห็นข้อมูลเชิงมูลค่าเพิ่มเติม)

### 📦 Materials (วัสดุ)
- ค้นหา / กรอง / แบ่งหน้ารายการวัสดุ
- **ค้นหาด้วยบาร์โค้ด:** สแกนบาร์โค้ดผู้ผลิต (EAN/UPC) หรือรหัสวัสดุของเราเองก็ได้ ระบบหาให้อัตโนมัติ
- เพิ่ม / แก้ไข / ปิดการใช้งานวัสดุ (soft delete — ข้อมูลเดิมยังอยู่ ไม่ถูกลบจริง)
- *สิทธิ์แก้ไข:* WAREHOUSE หรือ ADMIN

### 🏷️ หมวดหมู่วัสดุ (Categories)
จัดการหมวดหมู่ในหน้าเดียว: สร้าง / เปลี่ยนชื่อ (inline) / ลบ
- ทุกคนเห็นรายการได้ แต่ปุ่มแก้ไขจะแสดงเฉพาะ WAREHOUSE/ADMIN
- ลบหมวดหมู่ไม่ได้ถ้ายังมีวัสดุใช้หมวดนั้นอยู่ (กันข้อมูลกำพร้า)

### 📋 คำขอเบิกวัสดุ (Material Issues)
คำขอเบิกวัสดุไปใช้หน้างาน มีสถานะเป็นขั้นตอน (ดู [Workflow §4.1](#41-การเบิกวัสดุ))
- สร้างคำขอ: เพิ่มรายการด้วยการสแกนบาร์โค้ด หรือพิมพ์รหัส หรือใช้กล้องมือถือสแกน
- แถบสถานะ: รออนุมัติ / อนุมัติแล้ว / จ่ายแล้ว / จ่ายบางส่วน / ปฏิเสธ
- มีป้าย **"เกินกำหนด"** เตือนถ้าคำขอค้างเกิน SLA (รออนุมัติเกิน 24 ชม. หรืออนุมัติแล้วเกิน 3 วัน)

### 🚚 ใบสั่งซื้อ (Purchase Orders)
สั่งซื้อวัสดุจากซัพพลายเออร์
- สถานะ: ร่าง (DRAFT) → สั่งแล้ว (ORDERED) → รับบางส่วน/รับครบ (อัตโนมัติจากการรับของ) หรือ ยกเลิก (CANCELLED)
- *สิทธิ์สร้าง/จัดการ:* PURCHASING หรือ ADMIN

### 📥 รับวัสดุ (Goods Receive)
บันทึกการรับวัสดุเข้าคลัง
- **รับตาม PO:** เลือกคลัง + PO ระบบแสดงเฉพาะรายการที่ยังค้างรับ กรอกจำนวนที่รับ (รับเกินยอดสั่งไม่ได้)
- **รับแบบไม่มี PO:** เลือกซัพพลายเออร์แล้วเพิ่มรายการเอง
- *สิทธิ์:* WAREHOUSE (การรับเป็นงานคลัง แยกจากการสั่งซื้อ)

### 🔄 โอนย้ายระหว่างคลัง (Stock Transfers)
ย้ายวัสดุจากคลังหนึ่งไปอีกคลัง (เอกสาร `TR-`)
- เลือกคลังต้นทาง + ปลายทาง แล้วเพิ่มรายการ
- ระบบบันทึกคู่รายการออก/เข้าพร้อมกันแบบ atomic และ **ยกยอดต้นทุนเดิมติดไปด้วย** (ย้ายมูลค่า ไม่ใช่ตีราคาใหม่)
- *สิทธิ์สร้าง:* WAREHOUSE

### ✅ นับสต๊อก (Stock Counts)
นับจำนวนจริงเทียบกับระบบ แล้วปรับยอด (เอกสาร `SC-`)
1. เลือกคลัง → ระบบแสดงใบนับของวัสดุที่มีในคลังนั้น พร้อมยอดตามระบบ
2. กรอก **จำนวนจริง** ที่นับได้ (ค่าเริ่มต้น = ยอดระบบ)
3. รายการไหนไม่ตรงต้อง **ระบุเหตุผล**
4. ระบบปรับยอดเฉพาะรายการที่ต่าง (ลงบัญชี ADJUSTMENT)
- *สิทธิ์:* WAREHOUSE

### 🏢 โครงการ (Projects) — *เฉพาะ MANAGER/ADMIN*
จัดการโครงการ/ไซต์งาน
- ดูรายการโครงการ, งบประมาณ, มูลค่าสัญญา, คลังของโครงการ
- **สร้างโครงการใหม่:** สร้างโครงการ + ลูกค้า (เลือกเดิมหรือสร้างใหม่ในตัว) + คลังไซต์ พร้อมกันในครั้งเดียว
- **เปลี่ยนสถานะ:** วางแผน → กำลังดำเนินการ → เสร็จสิ้น (หรือ ยกเลิก)
- **ปิดโครงการปลอดภัย:** การกดปิด/ยกเลิกจะมี**ขั้นยืนยัน**ก่อนเสมอ (กดครั้งเดียวไม่ปิดทันที กันกดพลาด)
- **เปิดโครงการกลับ (Reopen):** ถ้าปิดผิด ADMIN สามารถเปิดโครงการที่ปิดไปแล้วกลับมาได้ (เฉพาะ ADMIN)
- โครงการที่ปิด/ยกเลิกแล้วจะ **เบิกวัสดุใหม่ไม่ได้**

### 📈 รายงาน (Reports)
แบ่งเป็น 2 กลุ่ม:

**รายงานทั่วไป** (ทุกคนดูได้)
- **สต๊อกคงเหลือ** — ยอดคงเหลือแต่ละวัสดุ/คลัง
- **วัสดุต่ำกว่าจุดสั่งซื้อ** — เตือนของใกล้หมด
- **กิจกรรมการเบิกจ่าย** — การเบิกในช่วงเวลา
- **กิจกรรมการรับวัสดุ** — การรับเข้าในช่วงเวลา
- **สัดส่วนสต๊อกตามไซต์** — กระจายสต๊อกตามคลัง/ไซต์

**รายงานการเงิน** (*เฉพาะ MANAGER/ADMIN*)
- **มูลค่าสต๊อก** — คำนวณต้นทุนถัวเฉลี่ยถ่วงน้ำหนักจริงจากประวัติทั้งหมด
- **ประวัติการเบิกจ่าย** — พร้อมมูลค่า
- **สรุปการเงินตามไซต์งาน** — คงเหลือ/เบิกใช้/รับเข้า ต่อไซต์

### 👥 ผู้ใช้งาน (Users) — *เฉพาะ ADMIN*
- เพิ่ม / แก้ไขผู้ใช้ กำหนด Role และ Access Level
- กำหนดคลัง/ไซต์ที่แต่ละคนเข้าถึงได้
- **ตั้งรหัสผ่านใหม่** ให้ผู้ใช้ (กรณีลืมรหัส)
- *กันล็อกตัวเอง:* ADMIN ลดสิทธิ์ตัวเองต่ำกว่า ADMIN หรือปิดบัญชีตัวเองไม่ได้

### 👤 บัญชีของฉัน (Account)
กดที่ชื่อ-รูปตัวเองมุมล่างซ้าย → ดูข้อมูลบัญชี และ **เปลี่ยนรหัสผ่านของตัวเอง** (ดู §6)

## 4. ขั้นตอนงานหลัก (Workflow)

### 4.1 การเบิกวัสดุ
```
REQUESTER สร้างคำขอ
      ↓ (รออนุมัติ PENDING_APPROVAL)
APPROVER อนุมัติ ──ปฏิเสธ──► REJECTED (จบ)
      ↓ (อนุมัติแล้ว APPROVED)
WAREHOUSE จ่ายวัสดุ
      ↓
FULFILLED (จ่ายครบ)  หรือ  PARTIALLY_FULFILLED (จ่ายบางส่วน — ของไม่พอ บันทึกยอดขาด)
```
- **ผู้ขอเบิกอนุมัติคำขอตัวเองไม่ได้** (ต้องเป็นคนอื่นอนุมัติ)
- จ่ายบางส่วนได้ ถ้าของในคลังไม่พอ ระบบจะบันทึกยอดที่ขาด (shortfall) ไว้

### 4.2 การจัดซื้อ → รับของ
```
PURCHASING สร้าง PO (ร่าง) → กดยืนยัน "สั่งแล้ว"
      ↓
WAREHOUSE รับของ (Goods Receive) อ้างอิง PO
      ↓
ระบบอัปเดตสถานะ PO อัตโนมัติ: รับบางส่วน / รับครบ
```
- รับเกินยอดสั่งไม่ได้
- รับของแบบไม่มี PO ก็ได้ (ต้องระบุซัพพลายเออร์)

### 4.3 หลักการบันทึกสต๊อก
ทุกการเคลื่อนไหวสต๊อก (รับ/จ่าย/โอน/ปรับ) จะถูกบันทึกเป็น **รายการบัญชีเดินสต๊อก (ledger)** ที่แก้ย้อนหลังไม่ได้ ยอดคงเหลือคำนวณจากผลรวมรายการเสมอ — ไม่มีการแก้ยอดตรงๆ ทำให้ตรวจสอบย้อนกลับได้ทุกครั้ง และ**สต๊อกติดลบไม่ได้**

## 5. กฎสำคัญที่ควรรู้

1. **จำนวนวัสดุต้องเป็นจำนวนเต็มเสมอ** (สั่งซื้อ/เบิก/รับ/โอน/นับ) — ใส่ทศนิยมไม่ได้ทุกช่อง
2. **จำนวนเงินใส่ทศนิยมได้** (ราคาทุน มูลค่าสัญญา งบประมาณ)
3. **STAFF ไม่เห็นราคาทุน/มูลค่า** ทุกที่ในระบบ
4. **สต๊อกติดลบไม่ได้** — จ่ายเกินยอดคงเหลือไม่ได้
5. **เอกสารเลขที่รันอัตโนมัติ** (MI- เบิก, PO- สั่งซื้อ, GR- รับของ, TR- โอน, SC- นับ) ต่อวัน
6. **การกระทำสำคัญกู้คืนได้:** ปิดโครงการมีขั้นยืนยัน + เปิดกลับได้ (ADMIN); ปิดวัสดุเป็น soft delete
7. **รหัสผ่านดูย้อนหลังไม่ได้** — เก็บแบบเข้ารหัสทางเดียว ทำได้แค่ "ตั้งใหม่" (ดู §6)

## 6. การจัดการรหัสผ่าน

### เปลี่ยนรหัสผ่านของตัวเอง (ทุกคนทำได้)
1. กดที่ชื่อตัวเองมุมล่างซ้าย → เข้าหน้า **บัญชีของฉัน**
2. กรอก **รหัสผ่านปัจจุบัน**, **รหัสใหม่** (อย่างน้อย 8 ตัว), และ **ยืนยันรหัสใหม่**
3. กด **เปลี่ยนรหัสผ่าน**

### แอดมินตั้งรหัสใหม่ให้ผู้อื่น (กรณีลืมรหัส)
1. ไปที่ **ผู้ใช้งาน** → เลือกผู้ใช้
2. ในส่วน **"ตั้งรหัสผ่านใหม่ให้ผู้ใช้นี้"** กรอกรหัสใหม่ → บันทึก
3. แจ้งรหัสใหม่ให้ผู้ใช้ แล้วแนะนำให้เปลี่ยนเองอีกครั้ง

> **หมายเหตุด้านความปลอดภัย:** ระบบ **ไม่สามารถแสดงรหัสผ่านเดิมของใครได้** แม้แต่ Admin — เพราะรหัสถูกเก็บแบบเข้ารหัสทางเดียว (bcrypt) ถอดกลับไม่ได้ ทำได้เพียง **ตั้งรหัสใหม่** ทับเท่านั้น ซึ่งครอบคลุมทุกกรณีที่ต้องช่วยผู้ใช้ที่เข้าระบบไม่ได้ และปลอดภัยกว่าการเก็บรหัสจริงไว้ (ถ้าฐานข้อมูลรั่ว รหัสทุกคนจะไม่หลุด)

## 7. คำถามที่พบบ่อย

**ถาม: เข้าระบบไม่ได้ ใส่รหัสผิดหลายครั้งแล้วถูกบล็อก?**
รอ ~15 นาทีแล้วลองใหม่ หรือให้ Admin ตั้งรหัสใหม่ให้

**ถาม: ทำไมไม่เห็นราคา/มูลค่าวัสดุ?**
บัญชีของคุณเป็นระดับ STAFF ซึ่งถูกออกแบบให้ไม่เห็นข้อมูลต้นทุน หากต้องเห็นให้ Admin ปรับเป็น MANAGER

**ถาม: กดปิดโครงการผิด ทำอย่างไร?**
แจ้ง Admin ให้ใช้ปุ่ม **เปิดโครงการกลับ (Reopen)** การปิดไม่ได้ลบข้อมูล เปิดกลับได้ทั้งหมด

**ถาม: จ่ายวัสดุแล้วของไม่พอ?**
ระบบให้จ่ายได้เท่าที่มี (จ่ายบางส่วน) และบันทึกยอดที่ขาดไว้ในคำขอ

**ถาม: เห็นเมนูไม่ครบเหมือนเพื่อน?**
เมนูแสดงตามสิทธิ์ เช่น "โครงการ" เห็นเฉพาะ MANAGER/ADMIN, "ผู้ใช้งาน" เห็นเฉพาะ ADMIN

**ถาม: พิมพ์เอกสารได้ไหม?**
ได้ หน้ารายละเอียดของคำขอเบิก / ใบสั่งซื้อ / รับของ / โอนย้าย / นับสต๊อก มีปุ่มดาวน์โหลด PDF สำหรับพิมพ์หรือบันทึก
