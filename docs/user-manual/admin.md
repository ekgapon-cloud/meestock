# คู่มือสำหรับ ผู้ดูแลระบบ (Admin)

> บทบาทของคุณ: **สิทธิ์สูงสุด** — ทำได้ทุกอย่างในระบบ และดูแลผู้ใช้/สิทธิ์
> เมนูที่คุณใช้: ทุกเมนู รวมถึง **ผู้ใช้งาน** ที่มีเฉพาะ ADMIN
> สิทธิ์ข้อมูล: ระดับ **ADMIN** — เห็นทุกคลัง เห็นต้นทุน และข้ามข้อจำกัด Role ได้ทั้งหมด

---

## เริ่มต้นใช้งาน
1. เปิดเว็บระบบ → กรอก **อีเมล** และ **รหัสผ่าน** → กด **เข้าสู่ระบบ**
2. เปลี่ยนรหัสผ่านตัวเอง: กดที่ชื่อตัวเองมุมล่างซ้าย → **บัญชีของฉัน**

> ในฐานะ ADMIN คุณทำงานของทุกบทบาทได้ (เบิก/อนุมัติ/รับ/จ่าย/โอน/นับ/จัดซื้อ/โครงการ) — ดูรายละเอียดแต่ละงานได้ในคู่มือของบทบาทนั้น หน้านี้เน้น **งานเฉพาะของ Admin**

---

## งานเฉพาะของ Admin

### 1) จัดการผู้ใช้งาน (เมนู ผู้ใช้งาน)
- **เพิ่มผู้ใช้ใหม่:** กำหนดชื่อ อีเมล รหัสผ่านเริ่มต้น
- **กำหนดสิทธิ์ 2 มิติ:**
  - **Role** (หน้าที่งาน): REQUESTER / APPROVER / WAREHOUSE / PURCHASING / EXECUTIVE
  - **Access Level** (ระดับข้อมูล): STAFF / MANAGER / ADMIN
- **กำหนดคลัง/ไซต์ที่เข้าถึงได้** ให้แต่ละคน (จำกัดการมองเห็นตามไซต์)
- **แก้ไข / ปิดใช้งานบัญชี**

### 2) ตั้งรหัสผ่านใหม่ให้ผู้ใช้ (กรณีลืมรหัส)
1. เมนู **ผู้ใช้งาน** → เลือกผู้ใช้
2. ส่วน **"ตั้งรหัสผ่านใหม่ให้ผู้ใช้นี้"** → กรอกรหัสใหม่ → บันทึก
3. แจ้งรหัสใหม่ให้ผู้ใช้ แล้วแนะนำให้เขาเปลี่ยนเองอีกครั้ง

> ⚠️ **ระบบแสดงรหัสผ่านเดิมของใครไม่ได้ แม้แต่ Admin** — รหัสถูกเก็บแบบเข้ารหัสทางเดียว (bcrypt) ถอดกลับไม่ได้ ทำได้เพียง **ตั้งรหัสใหม่ทับ** เท่านั้น ซึ่งครอบคลุมทุกกรณีช่วยผู้ใช้ที่เข้าระบบไม่ได้ และปลอดภัยกว่าการเก็บรหัสจริง (ถ้าฐานข้อมูลรั่ว รหัสทุกคนจะไม่หลุด)

### 3) เปิดโครงการที่ปิดไปแล้วกลับมา (Reopen) — เฉพาะ Admin
- โครงการที่ "เสร็จสิ้น/ยกเลิก" หากปิดผิด → เปิดหน้าโครงการ → กด **เปิดโครงการกลับ**
- โครงการกลับมาสถานะ "กำลังดำเนินการ" และเบิกวัสดุได้ตามปกติ (การปิดไม่ได้ลบข้อมูล จึงย้อนได้ครบ)

---

## ข้อควรรู้ (ความปลอดภัย)
- **กันล็อกตัวเอง:** คุณลดสิทธิ์ตัวเองต่ำกว่า ADMIN หรือปิดบัญชีตัวเองไม่ได้ (ป้องกันระบบไม่มี Admin เหลือ) — ในหน้าจอ ช่องเหล่านี้จะถูกปิดไว้เมื่อดูบัญชีตัวเอง
- **แยกหน้าที่ (Separation of Duties):** ระบบตั้งใจแยกฝ่ายจัดซื้อ (ออก PO) ออกจากคลัง (รับของ) — ควรกำหนด Role ให้ผู้ใช้ตามหน้าที่จริง ไม่ควรตั้งทุกคนเป็น ADMIN
- **STAFF ไม่เห็นต้นทุน** เป็นค่าเริ่มต้นที่ปลอดภัย — ให้ MANAGER เฉพาะผู้ที่ต้องเห็นข้อมูลการเงิน
- ทุกการเคลื่อนไหวสต๊อกถูกบันทึกเป็นบัญชีเดินสต๊อกที่ย้อนแก้ไม่ได้ — ตรวจสอบย้อนกลับได้เสมอ

---

## คำถามที่พบบ่อย
**อยากให้ผู้ใช้เห็นราคา?** ปรับ Access Level เป็น MANAGER
**พนักงานเห็นเมนูไม่ครบ?** ปกติ — เมนูแสดงตามสิทธิ์ (โครงการ = MANAGER/ADMIN, ผู้ใช้งาน = ADMIN)
**อยากจำกัดให้เห็นเฉพาะบางไซต์?** ตั้งค่า "คลัง/ไซต์ที่เข้าถึงได้" ในหน้าผู้ใช้ (ADMIN เห็นทุกไซต์เสมอ)
**ต้องดูแลระบบตอน deploy?** เครื่อง/container ที่ build เว็บควรมี RAM ว่างพอ (≥ 1 GB) ไม่งั้น Next.js อาจ prerender timeout
